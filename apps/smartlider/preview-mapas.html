🗺️ Funcionalidades implementadas

📍

GPS ao vivo

watchPositionAsync com Accuracy.BestForNavigation, atualiza a cada 2s ou 2m de deslocamento

🎯

Círculo de precisão

Anel azul proporcional à precisão GPS em metros, convertido para pixels com base no bbox do mapa

🔍

Zoom 0.5× – 4×

Botões + / − flutuantes sobre o mapa. ScrollView nested para pan livre em X e Y

⚠️

Detecção de área

Aviso laranja quando GPS sai do bbox do mapa (sw_lat/sw_lng/ne_lat/ne_lng)

⏺

Gravação de trajeto

Linha vermelha desenhada em tempo real. Mostra pontos, distância (Haversine) e duração. Salva em lider_trajetos

🗄️

GeoPDF → Supabase

Script Python extrai GPTS do PDF (ArcGIS/Avenza), converte para PNG e sobe na tabela lider_mapas

📐

Sem dependências extras

Linha desenhada com Views rotacionados (center-midpoint + atan2). Sem react-native-svg, sem WebView

🔒

Multi-workspace

Mapas e trajetos vinculados ao workspace_id. RLS garante isolamento total por site

⏭️ Próximos passos para colocar em produção

①
Rodar migration no Supabase
SQL Editor → colar conteúdo de supabase/migrations/add_lider_mapas.sql
②
Criar bucket de Storage
Supabase Storage → New bucket → nome: mapas-lider → Public ON
③
Processar os PDFs
python scripts/geopdf_to_supabase.py "PERDIZES_VIAS_BLOCO1.pdf" \
--workspace-id <uuid> --nome "Perdizes – Bloco 1" --tipo microplanejamento
④
Novo build EAS (obrigatório – expo-location é nativo)
cd "C:\smartpro-mobile\apps\smartlider"
git add -A ; git commit -m "feat: GPS map navigation"
npx eas build --platform android --profile preview
